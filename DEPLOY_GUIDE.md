# 微信云开发部署指南

## 问题说明

微信小程序使用**微信云开发**作为后端，不需要启动传统服务器，但需要完成以下配置：

1. ✅ 开通云开发服务
2. ✅ 创建云环境
3. ✅ 上传云函数
4. ✅ 创建数据库集合
5. ✅ 配置权限

---

## 第一步：开通云开发服务

### 1.1 在微信开发者工具中开通

1. 打开微信开发者工具
2. 点击工具栏上的 **"云开发"** 按钮
3. 如果是第一次使用，会提示开通云开发
4. 点击 **"开通"** 按钮
5. 填写环境名称（如：xiandu-dev）
6. 选择基础版（免费）
7. 点击确定

### 1.2 等待环境创建

- 环境创建需要 1-2 分钟
- 创建成功后会自动打开云开发控制台

## 第三步：上传云函数

### 3.1 查看云函数目录

在项目左侧文件列表中，找到 `cloudfunctions` 文件夹，包含以下云函数：

```
cloudfunctions/
├── login/              # 用户登录
├── addCheckin/         # 添加打卡
├── getMyCheckins/      # 获取打卡记录
├── getSpots/           # 获取点位列表
├── manageSpots/        # 点位管理
└── social/             # 社交功能
```

### 3.2 上传云函数

**方法一：逐个上传**
1. 右键点击 `cloudfunctions` 文件夹
2. 选择 **"同步云函数列表"**
3. 右键点击每个云函数文件夹（如 `login`）
4. 选择 **"上传并部署：云端安装依赖"**
5. 等待上传完成（每个约 30 秒）

**方法二：批量上传**
1. 右键点击 `cloudfunctions` 文件夹
2. 选择 **"同步云函数列表"**
3. 依次右键点击每个云函数上传

### 3.3 验证云函数

上传完成后，在云开发控制台：
1. 点击左侧 **"云函数"**
2. 查看是否显示所有云函数
3. 状态应该为 **"正常"**

## 第四步：配置权限

### 4.1 数据库权限设置

在云开发控制台的数据库中，为每个集合设置权限：

**users 集合**
- 权限设置：**仅创建者可读写**

**spots 集合**
- 权限设置：**所有用户可读，仅创建者可写**

**checkins 集合**
- 权限设置：**仅创建者可读写**

**comments 集合**
- 权限设置：**所有用户可读，仅创建者可写**

**favorites 集合**
- 权限设置：**仅创建者可读写**

**messages 集合**
- 权限设置：**仅创建者可读写**

### 4.2 设置方法

1. 点击集合名称
2. 点击 **"权限设置"** 标签
3. 选择对应的权限模板
4. 点击 **"保存"**

## 常见问题

### 问题1：云函数调用失败

**原因：** 云函数未上传或上传失败

**解决：**
1. 检查云函数是否已上传
2. 重新上传云函数
3. 查看云函数日志

### 问题2：数据库操作失败

**原因：** 集合未创建或权限不足

**解决：**
1. 检查集合是否已创建
2. 检查权限设置
3. 查看数据库日志

### 问题3：登录失败

**原因：** 云环境未初始化或 login 云函数未部署

**解决：**
1. 检查 app.js 中的云环境初始化
2. 确保 login 云函数已上传
3. 查看 login 云函数日志

### 问题4：地图不显示点位

**原因：** spots 集合中没有数据

**解决：**
1. 在 spots 集合中添加测试数据
2. 检查 getSpots 云函数是否正常
3. 查看控制台错误信息

### 问题5：打卡失败

**原因：** 距离验证或云函数问题

**解决：**
1. 检查距离验证逻辑（500米内）
2. 确保 addCheckin 云函数已上传
3. 查看云函数日志

## 环境配置说明

### 当前配置

您的项目使用的是动态环境配置：
```javascript
wx.cloud.init({
  env: wx.cloud.DYNAMIC_CURRENT_ENV,
  traceUser: true
})
```

这意味着：
- 在微信开发者工具中，会自动使用当前环境
- 不需要手动指定环境ID
- 更加灵活和安全

### 如果需要指定环境ID

如果您想指定具体的环境ID，可以修改 `app.js`：

```javascript
wx.cloud.init({
  env: 'your-env-id', // 替换为您的环境ID
  traceUser: true
})
```

环境ID可以在云开发控制台的 **"设置"** -> **"环境设置"** 中查看。
