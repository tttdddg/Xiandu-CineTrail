# 仙都影游打卡导览系统概述

## 1. 项目简介

### 1.1 项目背景

仙都影游打卡导览是一款面向丽水仙都景区的微信小程序，旨在为游客提供影视取景地和游戏IP关联点的打卡导览服务。通过整合景区文化资源与影视游戏IP，打造沉浸式文旅体验。

### 1.2 目标用户

- 仙都景区游客
- 影视剧爱好者
- 游戏玩家
- 文旅打卡爱好者

### 1.3 核心价值

- 🎬 影视取景地导览：探访热门影视剧拍摄地
- 🎮 游戏IP关联：体验游戏场景的现实原型
- 📍 智能打卡：记录旅行足迹，获取成就勋章
- 🗺️ 智能导览：地图导航、路线规划、附近推荐

## 2. 系统架构

### 2.1 技术栈

#### 前端技术

- **框架**: 微信小程序原生框架（WXML + WXSS + JavaScript）
- **地图服务**: 微信内置地图组件
- **UI设计**: 自定义组件库，遵循微信设计规范

#### 后端技术

- **云服务**: 微信云开发（CloudBase）
- **数据库**: 云开发数据库（NoSQL）
- **云函数**: Node.js 云函数

## 4. 数据库设计

### 4.1 用户表 (users)

```javascript
{
  _id: String,           // 用户ID
  openId: String,        // 微信OpenID
  nickName: String,      // 昵称
  avatarUrl: String,     // 头像URL
  role: Number,          // 角色（0-4）
  level: Number,         // 等级
  growthValue: Number,   // 成长值
  totalChecks: Number,   // 打卡总数
  continuousDays: Number,// 连续打卡天数
  lastCheckinDate: Date, // 最后打卡日期
  status: String,        // 状态
  createdAt: Date,       // 创建时间
  updatedAt: Date        // 更新时间
}
```

### 4.2 点位表 (spots)

```javascript
{
  _id: String,           // 点位ID
  name: String,          // 点位名称
  type: String,          // 类型（movie/game/culture/nature）
  typeName: String,      // 类型名称
  description: String,   // 详细描述
  coverImage: String,    // 封面图片URL
  latitude: Number,      // 纬度
  longitude: Number,     // 经度
  tags: Array,           // 标签列表
  hotScore: Number,      // 热度值（0-10）
  checkinCount: Number,  // 打卡总数
  likeCount: Number,     // 点赞数
  favoriteCount: Number, // 收藏数
  createdAt: Date,       // 创建时间
  updatedAt: Date        // 更新时间
}
```

### 4.3 打卡记录表 (checkins)

```javascript
{
  _id: String,           // 记录ID
  userId: String,        // 用户OpenID
  spotId: String,        // 点位ID
  photos: Array,         // 照片URL列表
  reviewStatus: String,  // 审核状态
  isMakeup: Boolean,     // 是否补签
  makeupReason: String,  // 补签原因
  location: Object,      // 打卡位置
  timestamp: Number,     // 时间戳
  createdAt: Date        // 创建时间
}
```

### 4.4 评论表 (comments)

```javascript
{
  _id: String,           // 评论ID
  userId: String,        // 用户OpenID
  spotId: String,        // 点位ID
  content: String,       // 评论内容
  likeCount: Number,     // 点赞数
  replyCount: Number,    // 回复数
  timestamp: Number,     // 时间戳
  createdAt: Date        // 创建时间
}
```

### 4.5 收藏表 (favorites)

```javascript
{
  _id: String,           // 收藏ID
  userId: String,        // 用户OpenID
  spotId: String,        // 点位ID
  createdAt: Date        // 创建时间
}
```

### 4.6 消息表 (messages)

```javascript
{
  _id: String,           // 消息ID
  userId: String,        // 用户OpenID
  type: String,          // 消息类型
  title: String,         // 标题
```

## 6. 权限设计

### 6.1 用户角色

| 角色ID | 角色名称   | 权限说明                 |
| ------ | ---------- | ------------------------ |
| 0      | 游客       | 仅可浏览公开内容         |
| 1      | 普通用户   | 可打卡、评论、点赞、收藏 |
| 2      | VIP用户    | 普通用户权限 + 专属功能  |
| 3      | 管理员     | 可管理点位、审核内容     |
| 4      | 超级管理员 | 拥有所有权限             |
